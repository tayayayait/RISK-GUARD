## [프로젝트 개요]

**프로젝트명**
RISK-GUARD

프로젝트 한 줄 소개
현장 담당자가 작업명, 작업 설명, 현장 사진만 입력하면, 한국산업안전보건공단(KOSHA) 공개데이터를 바탕으로 유사 재해사례, 사고사망 사례, 관련 법령·가이드, 교육자료를 자동 연결하여 위험성평가서 초안과 작업 전 안전 브리핑을 생성하는 웹 대시보드형 AI 서비스입니다.

서비스 핵심 가치
RISKOPE는 단순 검색형 시스템이 아니라, 여러 KOSHA 데이터를 한 화면에서 통합해 “왜 위험한지”와 “무엇을 해야 하는지”를 근거와 함께 제시하는 의사결정형 서비스입니다.
국내재해사례 게시판 정보 조회서비스는 유사 사고 패턴 탐색의 핵심 데이터원으로 활용되고, 사고사망 게시판 정보 조회서비스는 중대성 경고와 경각심 강화에 활용됩니다. 안전보건법령 스마트검색 API는 관련 법령과 KOSHA Guide를 찾는 데 활용되며, 안전보건자료 링크 서비스는 후속 교육자료와 안내자료를 추천하는 데 활용됩니다.
특히 Gemini API를 결합하여 사용자의 최소 입력만으로도 위험요인을 자동 추론하고, 공식 데이터 근거를 연결한 결과물을 생성한다는 점이 차별화 포인트입니다.

## [프로젝트 배경 및 목표]

프로젝트 배경
중소사업장이나 비전담 조직에서는 위험성평가를 작성해야 해도 어떤 사고사례를 참고해야 하는지, 어떤 법령을 적용해야 하는지, 어떤 교육자료를 함께 제공해야 하는지 한 번에 파악하기 어렵습니다. 실제로 관련 정보는 존재하지만, 재해사례는 재해사례대로, 사고사망 사례는 별도로, 법령과 자료도 각각 흩어져 있어 사용자가 이를 직접 검색하고 판단해야 하는 부담이 큽니다.

한국산업안전보건공단은 이미 국내재해사례 게시판, 사고사망 게시판, 안전보건자료 링크 서비스, 안전보건법령 스마트검색 등 다양한 공공데이터 API를 제공하고 있습니다. 그러나 현장에서는 이 데이터를 각각 찾아보는 데 시간이 많이 들고, 전문 용어와 법령 해석의 장벽 때문에 실제 활용도가 떨어질 수 있습니다.

이러한 문제를 해결하기 위해, 사용자는 최소한의 정보만 입력하고 AI가 나머지를 분석·구조화하며, KOSHA 공개데이터를 자동 연계해 위험성평가 작성 부담을 줄이고 품질은 높이는 서비스가 필요합니다.

프로젝트 목표
본 프로젝트의 목표는 다음과 같습니다.

첫째, 위험성평가 작성 시간을 줄여 현장 실무자의 부담을 낮추는 것입니다.
둘째, 단순한 서식 작성이 아니라 실제 재해사례와 법적 근거가 연결된 실효성 있는 위험성평가서를 생성하는 것입니다.
셋째, 작성으로 끝나는 문서가 아니라 교육자료와 안전 브리핑까지 연결되는 실행형 안전관리 흐름을 만드는 것입니다.
넷째, 작업 설명과 현장 사진 같은 비정형 입력을 Gemini API가 해석하여 업종, 작업장소, 사용 장비, 주요 위험요인 등을 자동 추론하고 정형화하는 것입니다.
다섯째, 공공데이터와 AI를 결합한 실제 현장 활용형 웹 대시보드를 구현하여 공모전의 핵심 평가 요소인 공공데이터 활용성, AI 활용성, 실현가능성, 효과성을 동시에 충족하는 것입니다.

## [상세 기능 요구 사항]

### 1. 작업 정보 입력 및 현장 상황 분석

사용자는 웹 대시보드에서 **작업명**과 **작업 설명**을 입력하고, 필요 시 **현장 사진**을 업로드합니다.
Gemini API는 입력된 텍스트와 이미지를 분석하여 **업종, 작업장소, 사용 장비, 주요 위험요인**을 자동 추론합니다. 이후 사용자는 드롭다운, 체크박스, 태그 입력 방식으로 AI가 제안한 결과를 확인하고 수정·확정할 수 있습니다.

이 단계에서 Gemini의 역할은 **“사진·자연어 입력을 구조화된 작업 프로필로 변환하는 것”**입니다. 이미지 이해 기능은 현장 사진 속 장비, 작업환경, 위험상황을 식별하는 데 활용되며, Structured Outputs 기능은 추출 결과를 표준화된 JSON 형태로 정리해 이후 재해사례 검색, 사고사망 사례 분석, 법령 검색, 교육자료 추천에 필요한 정규화된 조건을 생성하는 데 활용됩니다.

### 2. 유사 재해사례 자동 검색

**Gemini가 자동 추론하고 사용자가 최종 확정한 작업 프로필**을 바탕으로 국내재해사례 게시판 정보 조회서비스를 호출해 업종별 재해사례를 검색합니다.
이 서비스는 게시판 종류와 제목 키워드, 페이지 조건을 기준으로 재해사례를 조회할 수 있으므로, AI가 추론한 업종과 핵심 키워드를 기반으로 유사 사례를 수집하고 사용자가 확정한 위험요인을 반영해 검색 정확도를 높이는 구조로 설계합니다.

예를 들어 “지게차 상하차”, “고소작업대 점검”, “절단기 협착”, “전기설비 점검” 같은 키워드 조합을 자동 생성하여 유사 재해사례를 탐색할 수 있습니다.

**Gemini 적용 방식**
Gemini는 수집된 재해사례 원문을 단순 나열하는 대신, 여러 사례를 종합해 **반복 발생 위험, 사고 유발 조건, 즉시 점검 포인트** 형태로 요약합니다. 또한 표현은 다르지만 의미적으로 유사한 사례를 묶어 보여줌으로써 사용자가 빠르게 핵심 위험패턴을 이해할 수 있도록 지원합니다.

### 3. 중대사고 경고 및 심각도 보정

사고사망 게시판 정보 조회서비스를 활용하여 사망사고 일자, 장소, 사고개요, 사망자 수, 부상자 수 등 현재 작업과 유사한 중대사고 사례를 함께 조회합니다.
이 기능은 단순 참고용 사례 제시에 그치지 않고, **현재 작업의 위험등급을 조정하는 근거 데이터**로 활용됩니다. 즉, 동일하거나 유사한 위험요인이 실제 사망사고 사례와 연결되는 경우 위험등급을 상향 조정하는 구조입니다.

**Gemini 적용 방식**
Gemini는 사고개요를 읽고 이를 추락, 끼임, 붕괴, 질식, 폭발, 화학노출 등 표준 사고유형으로 분류합니다. 이후 현재 작업과의 유사도를 판단해 “이 작업은 최근 사망사고 패턴과 유사성이 높음”과 같은 설명형 경고문을 생성합니다. 이를 통해 단순 정보 제공이 아니라 **이유가 있는 위험 경고**를 제공할 수 있습니다.

### 4. 법령·KOSHA Guide 근거 자동 인용

안전보건법령 스마트검색 API를 활용하여 산업안전보건법령, 규칙, 고시, 훈령, 예규, 중대재해처벌법령, KOSHA Guide, 안전보건 미디어 자료 등을 검색합니다.
서비스는 작업 설명과 Gemini가 추출한 위험키워드를 질의어로 활용하여 관련 법령과 가이드를 찾고, 이를 위험성평가서의 **관련 근거 항목**으로 자동 삽입합니다.

전문 용어를 정확히 알지 못하더라도 현장 표현 기반의 검색이 가능하므로 현장 친화적인 법령 탐색이 가능합니다.

**Gemini 적용 방식**
Gemini는 검색된 법령과 가이드 중 현재 작업에 직접 관련된 항목만 선별하고, 이를 사람이 이해하기 쉬운 문장으로 재서술합니다. 예를 들어 “관련 법령”, “현장 적용 포인트”, “누락 시 위험”, “권장 조치”와 같은 형식으로 정리하여 단순한 법조문 나열이 아닌 **실무형 근거 정보**로 가공합니다.

### 5. 안전보건자료 자동 추천

안전보건자료 링크 서비스를 활용하여 책자, 안내서, OPS, 교안, 영상, 외국인용 자료 등 다양한 안전보건자료의 URL을 조회합니다.
위험성평가 결과가 생성된 후, 해당 업종과 주요 위험요인에 맞는 자료를 자동 추천하여 사용자가 후속 교육이나 작업 전 브리핑에 바로 활용할 수 있도록 합니다.

예를 들어 건설업 추락위험이 높은 경우에는 관련 영상과 OPS 자료를 우선 추천하고, 외국인 근로자가 포함된 현장인 경우에는 외국어 자료를 우선적으로 제시할 수 있습니다.

**Gemini 적용 방식**
Gemini는 추천 대상 자료를 요약해 “왜 이 자료를 봐야 하는지”를 한 줄 설명으로 제공하고, 작업유형·숙련도·언어환경을 고려해 자료 우선순위를 정렬합니다. 이를 통해 단순 링크 제공이 아닌 **상황 맞춤형 교육자료 추천**이 가능해집니다.

### 6. 위험성평가서 자동 생성

최종적으로 서비스는 작업 개요, 주요 위험요인, 유사 재해사례 요약, 사망사고 기반 중대위험 경고, 관련 법령 및 KOSHA Guide 근거, 권장 개선조치, 작업 전 점검 체크리스트, 추천 교육자료 링크가 포함된 **위험성평가서 초안**을 자동 생성합니다.
이 결과는 웹 대시보드 상에서 즉시 확인할 수 있으며, 복사 또는 파일 형태로 저장·출력할 수 있게 구성합니다.

위험성평가서에 포함되는 업종, 작업장소, 사용 장비, 주요 위험요인 정보는 **사용자의 최소 입력값 + Gemini의 자동 추론 결과 + 사용자의 최종 수정·확정 결과**를 반영합니다.

**Gemini 적용 방식**
Gemini는 여러 API의 응답과 사용자 입력을 종합하여 문장형 보고서와 구조화된 위험성평가 데이터를 동시에 생성합니다. 또한 작업 전 공유할 수 있는 짧은 안전 브리핑 문안까지 함께 생성하여, 문서 작성 이후의 현장 활용성까지 높입니다.

## [지원 디바이스 및 디자인]

**지원 디바이스**
본 서비스는 **웹 대시보드 전용**으로 구성합니다.
데스크톱과 노트북 브라우저 사용 환경을 기준으로 하며, 현장관리자, 안전담당자, 팀장 등이 사무실이나 현장사무소에서 활용하는 형태를 상정합니다. 모바일 앱은 본 프로젝트 범위에 포함하지 않습니다.

**디자인 방향**
화면은 한눈에 위험 상태와 근거 정보를 파악할 수 있도록 **대시보드 중심 구조**로 설계합니다.
메인 화면은 크게 네 개 영역으로 구성합니다.

좌측에는 작업 입력 패널을 배치하여 작업명, 작업 설명, 사진 업로드가 가능하도록 구성합니다.
중앙에는 AI 분석 결과를 배치하여 업종, 작업장소, 사용 장비, 주요 위험요인, 위험등급, 즉시 조치사항을 확인할 수 있게 합니다.
우측 상단에는 유사 재해사례와 사고사망 사례를 배치하여 실제 사고 근거를 보여줍니다.
우측 하단에는 관련 법령, KOSHA Guide, 교육자료 추천 영역을 배치하여 후속 조치와 학습 흐름까지 연결합니다.

사용자가 가장 먼저 봐야 하는 정보는 **현재 작업의 위험등급과 즉시 조치가 필요한 항목**이므로, 이를 상단 카드 형태로 강조합니다.
또한 법령과 사례는 원문 전체를 그대로 노출하기보다 **핵심 요약 + 근거 출처 펼침 구조**로 제공하여 검색형 서비스가 아니라 의사결정형 대시보드처럼 보이도록 구성합니다.

**권장 화면 구성**

1. **작업 입력 화면**: 작업명, 작업 설명, 사진 업로드
