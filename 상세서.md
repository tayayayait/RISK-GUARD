# RISK-GUARD 상세서

문서 버전: `v1.0`  
기준일: `2026-04-09`  
대상: 기획자, 디자이너, 프론트엔드 개발자, 백엔드 개발자

## 1. 문서 목적과 범위

본 문서는 `RISK-GUARD` 웹 대시보드의 구현 기준 문서다. 화면 구조, 사용자 흐름, 디자인 토큰, 컴포넌트 규격, 상태 정의, 반응형 규칙, 접근성 기준, 예외 처리 기준을 명확한 수치와 규칙으로 고정한다.

### 1.1 포함 범위

- 작업 입력, AI 분석, 근거 탐색, 교육자료 추천, 위험성평가서 출력까지의 전체 사용자 흐름
- KOSHA OpenAPI 4종과 Gemini 기반 구조화 결과를 소비하는 UI/UX 규칙
- 데스크톱 웹 대시보드 기준 정보 구조, 화면 배치, 상호작용 규칙
- 공통 디자인 시스템과 컴포넌트 설계 기준

### 1.2 제외 범위

- 모바일 앱
- 관리자용 통계 화면
- 다크 모드
- 다국어 UI 번역
- 사용자 권한 체계 세분화

### 1.3 기준 환경

| 항목 | 기준 |
|---|---|
| 지원 기기 | 데스크톱, 노트북 |
| 최소 지원 해상도 | `1024x768` |
| 권장 해상도 | `1440x900` 이상 |
| 기준 브라우저 | Chrome, Edge 최신 2개 버전 |
| 방향 | 가로형 레이아웃 고정 |
| 테마 | 라이트 테마 단일 운영 |

## 2. 서비스 구조

### 2.1 화면 구조와 라우팅

서비스는 단일 대시보드 셸 안에서 6개 화면을 단계형으로 이동하는 구조로 구현한다.

| 화면 ID | 라우트 | 화면명 | 목적 | 다음 이동 조건 |
|---|---|---|---|---|
| `SCR-01` | `/assessments/new` | 작업 입력 | 작업명, 설명, 사진 입력 | 필수값 유효성 통과 후 `AI 분석 시작` |
| `SCR-02` | `/assessments/:id/profile-review` | AI 분석 확인 | Gemini 추론 결과 검토 및 수정 | 필수 프로필 확정 |
| `SCR-03` | `/assessments/:id/analysis` | 분석 결과 | 위험등급, 즉시 조치, 사고 시나리오 확인 | 위험 분석 데이터 생성 완료 |
| `SCR-04` | `/assessments/:id/evidence` | 근거 화면 | 유사 재해사례, 사고사망 사례, 법령·Guide 확인 | 근거 선택 또는 검토 완료 |
| `SCR-05` | `/assessments/:id/materials` | 교육 화면 | 안전보건자료 추천 및 선택 | 자료 검토 완료 |
| `SCR-06` | `/assessments/:id/report` | 문서 출력 | 위험성평가서 초안, 체크리스트, 브리핑 문안 편집 및 저장 | 내보내기 성공 또는 복사 완료 |

### 2.2 데이터 소스 매핑

| 소스 | 사용 화면 | 입력값 | 출력값 | UI 표시 방식 |
|---|---|---|---|---|
| Gemini 텍스트/이미지 분석 | `SCR-02`, `SCR-03`, `SCR-06` | 작업명, 설명, 사진 | 업종, 작업장소, 장비, 위험요인, 즉시 조치, 브리핑 문안 | 구조화 카드, 태그, 경고문, 문서 초안 |
| KOSHA 국내재해사례 API | `SCR-04`, `SCR-06` | 업종, 키워드, 페이지 | 유사 재해사례 목록 | 사례 카드, 요약 불릿, 인용 근거 |
| KOSHA 사고사망 API | `SCR-03`, `SCR-04`, `SCR-06` | 위험유형, 작업장소, 키워드 | 사망사고 일자, 장소, 개요, 피해 규모 | 중대위험 경고 카드, 사례 카드 |
| KOSHA 법령 스마트검색 API | `SCR-04`, `SCR-06` | 위험키워드, 작업명 | 법령, 고시, Guide, 미디어 자료 | 근거 카드, 적용 포인트 리스트 |
| KOSHA 안전보건자료 링크 API | `SCR-05`, `SCR-06` | 업종, 재해유형, 자료형태, 외국어 구분 | 자료 URL, 제목, 분류 | 추천 카드, 링크 버튼, 브리핑 첨부 |

### 2.3 결과 수집 및 기본 노출 수

| 데이터 영역 | 수집 최대값 | 최초 노출 수 | 더보기 단위 | 정렬 기준 |
|---|---|---|---|---|
| 유사 재해사례 | `20건` | `5건` | `5건` | 관련도, 최신순 |
| 사고사망 사례 | `10건` | `3건` | `3건` | 관련도, 최신순 |
| 법령·Guide | `12건` | `6건` | `3건` | 관련도 우선 |
| 교육자료 | `12건` | `8건` | `4건` | 관련도, 자료형 우선순위 |
| 보고서 인용 근거 | `12건` | 전체 | 없음 | 사용자가 선택한 순서 |

### 2.4 검색어 생성 규칙

- 국내재해사례 검색은 `업종 1개 + 핵심 위험요인 2개 + 장비명 1개` 조합으로 최대 `4개 질의`를 만든다.
- 사고사망 검색은 `위험요인 1순위`, `작업장소`, `장비명`을 조합한 `2개 질의`를 만든다.
- 법령 검색은 `작업명`, `위험요인`, `조치 키워드`를 묶은 `3개 질의`를 만든다.
- 교육자료 검색은 `업종`, `재해유형`, `자료형태`, `외국어 여부`를 사용한다.
- 중복 제거 기준은 `제목 + 발행일` 또는 `제목 + URL`이다.

## 3. 전역 레이아웃 규칙

### 3.1 기본 셸

`SCR-02`부터 `SCR-06`까지는 동일한 대시보드 셸을 사용한다.

| 영역 | 규격 | 규칙 |
|---|---|---|
| 상단 헤더 | 높이 `64px` | 로고, 현재 단계명, 저장 상태, 우측 전역 액션 배치 |
| 좌측 단계 레일 | 폭 `240px` | 6단계 스테퍼 고정, 현재 단계 강조 |
| 메인 콘텐츠 | 가변 `1fr` | 최대 폭 제한 없음, 내부 12컬럼 그리드 사용 |
| 우측 유틸리티 레일 | 폭 `320px` | 요약 정보, 선택 근거, 저장 상태 표시 |
| 페이지 내부 패딩 | 좌우 `24px`, 상하 `24px` | `>=1600px`에서는 좌우 `32px` |
| 카드 간격 | `16px` 기본, 섹션 간 `24px` | 동일 깊이 카드 간격은 항상 동일 유지 |

- 본문 배경은 항상 `color-neutral-050`를 사용한다.

- 테두리는 `1px solid color-neutral-200`를 기본값으로 사용한다.
- 경고 정보는 색상만으로 구분하지 않고 아이콘과 텍스트 라벨을 반드시 함께 사용한다.
- `color-accent-600`은 CTA 배경으로 사용하지 않는다. CTA는 `color-primary-700` 고정이다.

## 6. 공통 컴포넌트 규격

### 6.1 버튼

| 항목 | `lg` | `md` | `sm` |
|---|---|---|---|
| 높이 | `48px` | `40px` | `32px` |
| 좌우 패딩 | `20px` | `16px` | `12px` |
| 아이콘 크기 | `18px` | `16px` | `16px` |
| 모서리 | `12px` | `12px` | `8px` |

| variant | 배경 | 텍스트 | 테두리 | 사용처 |
|---|---|---|---|---|
| `primary` | `color-primary-700` | `#FFFFFF` | 없음 | 단계 진행, 저장, 분석 시작 |
| `secondary` | `#FFFFFF` | `color-neutral-900` | `color-neutral-300` | 보조 액션 |
| `ghost` | 투명 | `color-neutral-700` | 없음 | 카드 내부 보조 액션 |
| `danger` | `color-danger-600` | `#FFFFFF` | 없음 | 삭제, 제외 |

버튼 상태 규칙:

- hover 시 배경은 한 단계 진하게 변경한다.
- disabled 시 배경은 `color-neutral-100`, 텍스트는 `color-neutral-500`, 커서는 `not-allowed`다.
- loading 시 텍스트를 숨기지 않고 좌측에 스피너 `16px`를 추가한다.

### 6.2 카드

| 유형 | 패딩 | 최소 높이 | 용도 |
|---|---|---|---|
| `section-card` | `20px` | 없음 | 일반 정보 블록 |
| `metric-card` | `20px` | `132px` | 점수, 건수 |
| `evidence-card` | `20px` | `168px` | 사례, 법령, 자료 카드 |
| `alert-card` | `20px` | `140px` | 즉시 조치, 치명 경고 |
| `report-card` | `24px` | 없음 | 문서 편집 |

카드 규칙:

- 기본 카드 테두리는 `1px solid color-neutral-200`다.
- `alert-card`는 좌측 `4px` 강조바를 사용한다.
- 카드 내부 제목과 본문 간격은 `8px`, 본문과 액션 간격은 `16px`다.

### 6.3 입력창

| 컴포넌트 | 높이 | 규칙 |
|---|---|---|
| 텍스트 입력 | `44px` | 라벨 상단 배치, placeholder 최대 30자 |
| 텍스트영역 | 최소 `132px` | 자동 세로 확장 허용, 최대 높이 `280px` |
| 셀렉트 | `44px` | 단일 선택, 검색형 허용 |
| 멀티태그 입력 | `44px` 이상 | 태그 `8개`까지, 태그당 최대 `16자` |
| 체크박스 | `20px` | 라벨 클릭 영역 전체 활성화 |
| 토글 | `40x24px` | 이진 상태에만 사용 |

입력 상태:

| 상태 | 테두리 | 배경 | 추가 요소 |
|---|---|---|---|
| 기본 | `color-neutral-300` | `#FFFFFF` | 없음 |
| focus | `color-primary-600` | `#FFFFFF` | 외곽 `4px` focus ring |
| error | `color-danger-600` | `#FFFFFF` | 하단 오류문구 |
| disabled | `color-neutral-200` | `color-neutral-100` | 텍스트 `color-neutral-500` |

### 6.4 파일 업로드

| 항목 | 규격 |
|---|---|
| 드롭존 높이 | `180px` |
| 드롭존 테두리 | `1.5px dashed color-neutral-300` |
| 허용 포맷 | `jpg`, `jpeg`, `png`, `webp` |
| 개수 제한 | 최대 `5장` |
| 파일 크기 | 장당 `10MB` 이하 |
| 권장 해상도 | `1280x720` 이상 |

업로드 규칙:

- 업로드 직후 썸네일을 `96x96px`로 표시한다.
- 업로드 중인 파일은 썸네일 위에 `progress bar`를 표시한다.
- 실패 파일은 썸네일을 남기지 않고 오류 토스트를 띄운다.

### 6.5 탭, 아코디언, 배지

| 컴포넌트 | 규격 | 규칙 |

| 태그 칩 | 높이 `28px` | 장비, 위험요인, 검색어 편집용 |

### 6.6 모달, 드로어, 토스트

| 컴포넌트 | 폭 | 규칙 |
|---|---|---|
| 확인 모달 | `480px` | 삭제, 이동 이탈 확인 |

- 오류 토스트는 자동 닫힘 없음
- 동일 오류 메시지는 `1회`만 띄운다

### 7.2 사용자 표시 상태

| `error` | 처리 실패 | 재시도 가능 상태 |

### 7.3 신뢰도 구간

| 신뢰도 | 기준 | UI 처리 |
|---|---|---|
| `high` | `0.85 이상` | 기본 표시 |
| `medium` | `0.65 이상 ~ 0.84 이하` | 정보 배지 표시 |
| `low` | `0.64 이하` | 빨간 점선 테두리와 확인 배너 표시 |

- 신뢰도는 발생확률이 아니라 AI 추론의 근거 충실도 지표다.

### 7.4 위험점수 계산 규칙

위험점수는 `0~100` 정수로 계산한다.

1. 확인된 위험요인 최대 `3개`를 선택한다.  
2. 각 위험요인 가중치를 합산한다.  
3. 고위험 장비 보정치를 더한다.  
4. 사고사망 사례 유사도 보정치를 더한다.  
5. 최종값은 `100`을 초과할 수 없다.

위험요인 가중치:

| 위험유형 | 점수 |
|---|---|
| 추락 | `30` |
| 붕괴 | `35` |
| 질식 | `35` |
| 폭발/화재 | `35` |
| 감전 | `35` |
| 끼임/협착 | `25` |
| 절단 | `25` |
| 낙하물/비래 | `20` |
| 차량/이동장비 충돌 | `25` |
| 화학노출 | `25` |
| 소음/분진/반복작업 | `10` |

고위험 장비 보정:

- 지게차, 크레인, 고소작업대, 절단기, 배전반, 밀폐공간 장비 포함 시 `+15`

사고사망 유사도 보정:

- 유사도 `0.80 이상` 사례 존재 시 `+20`
- 유사도 `0.60 이상 ~ 0.79 이하` 사례 존재 시 `+10`

위험등급 매핑:

| 점수 | 위험등급 |
|---|---|
| `90 이상` | `critical` |
| `70~89` | `high` |
| `40~69` | `medium` |
| `0~39` | `low` |

- 진행 문구 3단계 순차 표시
- `텍스트 분석 중` → `사진 분석 중` → `작업 프로필 정리 중`

오류 규칙:

- 텍스트 길이 부족 시 입력 하단에 즉시 오류 표시
- 업로드 실패 시 해당 파일명을 포함한 오류 토스트 표시

| 컬럼 | 설명 |
|---|---|
| 위험요인명 | 사용자 수정 가능, 행 내부 보조 버튼으로 삭제 |

신뢰도 규칙:

- 각 AI 추론값 우측에 `confidence badge` 표시
- `low` 신뢰도 항목은 빨간 점선 테두리와 `확인 필요` 라벨 표시
- `low` 항목이 1개라도 남아 있으면 상단 배너 표시
- 신뢰도 표시는 발생확률 용도가 아니다.

진행 조건:

- `industry`, `workLocation`, `equipment`, `hazards` 확정 시에만 `분석 결과 확정` 활성화
- 재분석은 기존 수정값을 버리고 `SCR-01` 입력값 기준으로 다시 실행

- `RISK-GUARD_{taskName}_{YYYYMMDD}.docx`

내보내기 규칙:

- `taskName`, `industry`, `hazards`, `riskLevel` 없으면 내보내기 불가
- 근거 데이터 일부 누락 시에도 내보내기는 가능하되 누락 섹션에 `근거 수집 실패` 문구 삽입

## 9. 사용자 흐름

### 9.1 기본 흐름

1. 사용자가 `SCR-01`에서 작업명과 설명을 입력한다.
2. 필요 시 현장 사진을 업로드한다.
3. `AI 분석 시작`을 누른다.
4. 시스템이 Gemini로 작업 프로필을 생성한다.
5. 사용자가 `SCR-02`에서 업종, 작업장소, 장비, 위험요인을 확정한다.
6. 시스템이 위험점수와 위험등급을 계산하고 `SCR-03`을 연다.
7. 시스템이 KOSHA API 4종을 병렬 호출한다.
8. 사용자가 `SCR-04`에서 근거를 검토하고 필요한 항목을 인용 목록에 담는다.
9. 사용자가 `SCR-05`에서 교육자료를 선택한다.
10. 시스템이 `SCR-06`에서 위험성평가서 초안을 생성한다.
11. 사용자가 문서를 수정하고 저장 또는 복사한다.

### 9.2 대체 흐름

| 상황 | 처리 |
|---|---|
| 사진 없이 시작 | 텍스트만으로 분석 진행 |
| AI 결과가 부정확 | `SCR-02`에서 수동 수정 후 확정 |
| 재해사례 없음 | 넓은 키워드 검색으로 1회 자동 재시도 |
| 사고사망 사례 없음 | `유사 사망사고 미검출` 메시지 표시, 위험등급 유지 |
| 법령 검색 실패 | 사례와 AI 추론만으로 보고서 생성 허용 |
| 내보내기 실패 | 편집 내용 유지, 재시도 버튼 제공 |

## 10. 반응형 기준

### 10.1 브레이크포인트

| 구간 | 범위 | 동작 |
|---|---|---|
| `desktop-xl` | `1600px 이상` | 3영역 전체 노출 |
| `desktop` | `1280px~1599px` | 기본 레이아웃 |
| `desktop-sm` | `1024px~1279px` | 우측 레일을 드로어로 전환 |
| `unsupported` | `1023px 이하` | 서비스 이용 제한 안내 |

### 10.2 구간별 규칙

`desktop-xl`

- 좌측 레일 `264px`
- 우측 레일 `360px`
- 카드 2열 그리드 허용

`desktop`

- 좌측 레일 `240px`
- 우측 레일 `320px`
- 메인 카드 1열 기본

`desktop-sm`

- 좌측 레일 `72px` 아이콘형 축소
- 우측 레일은 버튼 클릭 시 드로어 오픈
- 근거 화면 탭 내부 카드 간격 `12px`

`unsupported`

- 전체 앱 대신 차단 화면 노출
- 메시지: `RISK-GUARD는 1024px 이상 데스크톱 환경에서 사용하도록 설계되었습니다.`

## 11. 접근성 원칙

### 11.1 필수 기준

- WCAG `2.2 AA` 수준 준수
- 일반 텍스트 대비비 `4.5:1` 이상
- 큰 텍스트 대비비 `3:1` 이상
- 키보드만으로 전 화면 이동 가능
- 모달 오픈 시 focus trap 필수

### 11.2 키보드 상호작용

| 컴포넌트 | 규칙 |
|---|---|
| 스테퍼 | `Tab`, `Shift+Tab`, `Enter`로 이동 |
| 탭 | `좌우 화살표`로 전환 |
| 태그 입력 | `Backspace`로 마지막 태그 삭제 |
| 모달 | `Esc` 닫기 가능 |
| 업로드 | 키보드 포커스로 파일 선택 가능 |

### 11.3 스크린리더 규칙

- 위험등급 배지는 `위험등급: 높음`처럼 완전한 문장으로 읽혀야 한다.
- 로딩 상태는 `aria-live="polite"`로 공지한다.
- 오류 메시지는 입력 필드와 `aria-describedby`로 연결한다.
- 아이콘 단독 버튼은 모두 `aria-label`을 가진다.

### 11.4 접근성 보조 규칙

- 필수 입력은 색상 대신 `필수` 텍스트 라벨을 함께 제공한다.
- 결과 없음 상태는 빈 영역 대신 이유와 다음 행동을 함께 제공한다.
- 링크는 모두 `새 창 열림` 여부를 시각적으로 표시한다.

## 12. 예외 처리 기준

| 예외 상황 | 감지 조건 | UI 처리 | 후속 동작 |
|---|---|---|---|
| Gemini 분석 타임아웃 | `45초` 초과 | 상단 오류 배너 + 재시도 버튼 | 입력값 유지 |
| 이미지 포맷 오류 | 허용 포맷 아님 | 업로드 즉시 차단 | 파일 재선택 |

| KOSHA API 일부 실패 | 4개 중 일부 실패 | 실패한 카드만 `partial` 상태 | 나머지 결과 유지 |
| 결과 없음 | `ApiStatus=empty` | 빈 상태 일러스트 대신 문장 + 검색 확장 제안 | 자동 재검색 1회 |
| 세션 만료 | 인증 실패 또는 저장 실패 | 전체 화면 세션 만료 모달 | 재로그인 후 초안 복구 |

| 파일 내보내기 실패 | PDF/DOCX 생성 실패 | 토스트 + 오류 배너 | 재시도 |
| 근거 0건인데 보고서 생성 | 인용 목록 비어 있음 | 경고 배너 표시 | 생성은 허용 |

## 13. 구현 체크포인트

- 위험등급은 색상, 아이콘, 라벨 3요소를 동시에 사용해야 한다.
- 모든 카드에는 출처 또는 데이터 상태가 보여야 한다.
- `SCR-02` 이전에는 사용자가 AI 결과를 신뢰하고 넘어가도록 유도하지 않는다. 필수 확정 단계는 제거 금지다.
- `SCR-04`와 `SCR-05`는 단순 링크 나열이 아니라 `왜 추천되는지` 설명이 반드시 들어가야 한다.
- `SCR-06`은 읽기 전용 미리보기 화면이 아니라 수정 가능한 문서 작성 화면이어야 한다.
