# 예외 처리 규칙

## 기본 원칙

- 실데이터만 사용한다.
- 근거 데이터(mock/샘플) 자동 대체는 금지한다.
- 실패와 빈결과를 구분해 상태를 표시한다.

## 시나리오별 처리

- Gemini 분석 실패
  - 재시도 후 실패 시 에러 반환
  - 텍스트 모델은 요청 단위 자동 라우팅으로 선택한다.
  - 기본은 `gemini-3-flash-preview`(비용 우선), 고복잡도(장문/구조화 지시 다수/멀티모달 장문)는 `gemini-3.1-pro-preview`를 우선 선택한다.
  - `VITE_GEMINI_TEXT_MODEL`이 flash/pro 외 커스텀 모델이면 해당 모델을 먼저 시도한 뒤 flash/pro 후보로 폴백한다.
  - `generateContent`에서 모델 미존재(404) 또는 재시도 가능 오류(429/500/503/504) 발생 시 다음 후보 모델로 자동 재시도한다.
  - 모든 후보가 실패하면 설정 오류로 종료 (`VITE_GEMINI_TEXT_MODEL` 확인 필요)
  - UI: "목업 데이터 미사용" 메시지 표시
  - 시나리오 생성 전 `입력 인식 단계`를 내부 자동으로 수행하고, KOSHA CSV(`기계설비명 + 기계설비설명`) 후보를 우선 사용
  - 인식 단계는 Gemini를 항상 호출하되, 응답 파싱 실패/결측 시 규칙 점수 1위 후보로 안전 폴백
  - 사고 시나리오 응답은 JSON 파싱을 우선 시도하고, 코드블록/불완전 응답은 정리 후 재파싱
  - JSON 파싱 실패 또는 필드 누락 시에도 클라이언트 정규화 폴백으로 시나리오 카드 3개를 강제 보장

- Gemini 이미지 생성 실패
  - 클라이언트 이미지 생성 호출은 `models/{model}:generateContent` 엔드포인트만 사용
  - 업로드 이미지가 있으면 이미지 생성 요청 `parts`를 `텍스트 지시문 + inlineData(참조 이미지)`로 구성해 현장 맥락(설비/배치/작업조건/구도)을 최우선으로 고정
  - 이미지 지시문은 인식 단계의 `canonicalEquipment / operationContext / hazardParts / sceneConstraints`를 하드 제약으로 사용
  - 이미지 지시문 우선순위는 `참조 이미지 > 장비/현장 설명 > 위험 지점/위험 행동 > 사고유형(보조 라벨)`로 유지
  - 사고유형이 장비/현장 맥락과 충돌하면 사고유형을 맥락 범위 안에서 재해석하고, 맥락 자체는 변경하지 않음
  - 사고유형별 위험성은 `보조 라벨` 수준에서 시각 신호(끼임 간극, 추락 불안정 자세, 감전 접촉 근접, 비래물 궤적 등)를 강화해 한눈에 인지되게 구성
  - 사고유형 정규화는 18종 표준 키(`끼임/협착/절단/베임/찔림/충돌/부딪힘/맞음/낙하·비래/추락/넘어짐/감전/화상/폭발/화재/깔림/압궤/붕괴`)를 기준으로 수행하고, 한/영 동의어를 동일 키로 매핑
  - 생성 후 품질 게이트는 `휴리스틱 1차 + Gemini 이미지검증(경계 케이스만)`의 혼합 방식으로 동작
  - 휴리스틱 1차 품질 기준은 `위험원 + 작업자 노출경로 + 사고 진행 방향 + 즉시조치 + 부상위험 부위` 5요소로 계산하고, 하드 게이트 보조 플래그를 함께 추론
  - 품질 미달(`soft_fail`) 시 강화 프롬프트로 최대 2회 자동 재생성하고 재판정
  - 하드 게이트 7축(`equipmentContextAligned / mechanismSalienceVisible / typeDiscriminatorVisible / hazardHotspotSalienceVisible / injuryBodyPartEmphasisVisible / trajectoryVectorVisible / immediateActionPointVisible`) 중 하나라도 `false`면 `soft_fail` 처리
  - 하드 게이트 미달 항목은 항목별 보강 힌트를 재생성 프롬프트 최우선으로 주입
  - `부상위험 부위`가 프레임에서 불명확하면 `soft_fail`로 처리하고, 위험원-신체부위 임박 경로를 보강 힌트로 재주입
  - `핵심 위험 지점`, `부상 가능 부위`, `위험 진행 방향`, `즉시 조치 포인트` 4신호는 프롬프트에서 필수 시각신호 블록으로 고정
  - 재판정 후에도 품질 미달이면 최선 이미지를 카드 화면에는 그대로 표시하며 추가 경고 UI는 노출하지 않음
  - 수동크레인/체인블록 계열은 체인·훅·풀리 가시성, 원경 금지, 장비 프레임 점유 제약을 우선 적용
  - 절단/재단 계열은 파편(칩/톱날 조각) 비산 궤적과 부재 킥백(밀림/튕김) 순간이 프레임에서 명시적으로 보여야 함
  - 위험 장면은 정적 작업 스냅샷이 아니라 `위험원 + 작업자 노출 경로 + 충돌/비산 방향`이 동시에 보이도록 강제
  - 참조 이미지 기반 생성 시 무관한 업종/배경/기종 전환을 금지하고, 결과 표현은 `근접사고(near-miss)` 중심으로 제한
  - 결과 스타일은 포토리얼 정사진 대신 산업안전 교육용 반실사 일러스트를 기본으로 하되, 설비/축척/원근은 실제 현장과 정합하게 유지
  - 시나리오 카드는 클릭 시 개별 이미지 생성(지연 생성)하며, 카드별로 로딩/실패 상태를 독립 관리
  - 특정 카드 이미지 생성 실패는 해당 카드에만 에러를 표시하고 다른 카드/텍스트 시나리오는 유지
  - 게시용 파일 생성 시 시나리오 이미지를 순차 확보하고 `qualityStatus=pass` 항목만 단일 포스터 캔버스로 병합
  - `soft_fail` 항목은 번들(PNG/PDF)에서 자동 제외하고, 다운로드 완료 토스트로 제외 사실을 안내
  - 통합 PNG/PDF 생성 단계에서 특정 시나리오 이미지가 누락되면 파일 생성을 중단하고 실패 사유를 표시
  - UI/포스터 오버레이는 다중 박스·라인 강조를 사용하지 않고 `위험 구간 확대` 인셋만 유지
  - 이미지 내부 `Generated by Gemini` 워터마크 문구는 표시하지 않음
  - 업로드 이미지가 없으면 기존 텍스트 기반 이미지 생성 경로를 유지
  - 모델 404 발생 시 `VITE_GEMINI_IMAGE_MODEL -> gemini-3.1-flash-image-preview -> gemini-2.5-flash-image -> gemini-3-pro-image-preview` 순서로 자동 재시도
  - 응답에 `candidates[].content.parts[].inlineData`가 없으면 이미지 미생성으로 처리하고 텍스트 리포트만 제공

- KOSHA 근거 API 일부/전체 실패
  - 해당 탭 상태를 `error`로 표시
  - 실패 탭은 빈 목록 유지
  - 다른 탭 성공 결과는 유지

- KOSHA 근거 API empty
  - 해당 탭 상태를 `empty`로 표시
  - UI에 "결과 없음" 안내

- 근거 0건 보고서 생성
  - 생성은 허용
  - 경고 배너 및 신뢰도 저하 안내 문구 표시

- 내보내기 실패
  - `reportExportState[format]=error`
  - 오류 메시지 노출 및 재시도 허용

## 근거 매칭 엔진 폴백

- LLM 재랭킹 기본 ON
- 재랭킹 타임아웃 1.5초
- 타임아웃/응답 파싱 실패 시 규칙 점수만으로 계속 진행

## 입력/파일 제한

- 업로드 허용 확장자: `jpg`, `jpeg`, `png`, `webp`
- 파일 최대 용량: `10MB`
- 최대 파일 수: `5`
- 업로드 입력원: 로컬 파일 드래그/선택만 지원 (`text/uri-list` 형태의 웹 이미지 URL 드래그는 비지원)
