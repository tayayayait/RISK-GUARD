# 보고서 내보내기 프로필

## 목적
- 문서 출력 기본값을 `제출용(submission)`으로 고정한다.
- `검토용(review)`은 별도 확인 가능하도록 유지한다.
- 편집 데이터는 하나의 소스를 공유하고, 내보내기 시점에만 프로필별 섹션 조합을 적용한다.

## 기본값
- 기본 프로필: `submission`
- 내보내기 대상: `PDF`, `DOCX`, `Clipboard` 모두 현재 선택 프로필 기준
- 파일명 규칙: `RISK-GUARD_{task}_{yyyymmdd}_{profile}.{ext}`
  - `profile`: `submission` | `review`

## 프로필별 섹션 매핑

### submission (기본, 제출용)
- 작업 개요 (`overview`)
- 주요 위험요인 (`hazards`)
- 위험등급 및 즉시 조치 (`risk-level`)
- 조치계획 (`action-plan`)
  - `법령 기반 개선조치(law-remedial-actions)` + `권장 개선조치(improvements)` 통합
- 법령·가이드 근거 요약 (`law-guide-summary`)
  - `법령/Guide/미디어` 인용만 유지
  - 법령/Guide 인용 항목 중 `AI 요약`이 생성된 경우에만 `우리 회사 사고와의 관련성 / 적용 이유 / 실제 조치` 3개 섹션을 함께 출력
  - `교육자료 링크` 구간은 제외
- 작업 전 체크리스트
- 작업 전 안전 브리핑 문안

### review (검토용)
- 기존 전체 섹션을 유지한다.
- 출력 구조만 `본문 + 부록`으로 재정렬한다.
- 본문:
  - 작업 개요, 작업 프로필, 주요 위험요인, 위험등급 및 즉시 조치, 법령 기반 개선조치, 권장 개선조치
- 부록:
  - 유사 재해사례 요약, 사망사고 기반 경고, 법령 및 KOSHA Guide 근거, 추천 교육자료
- 체크리스트/브리핑은 본문 뒤 공통 섹션으로 유지한다.

## 구현 원칙
- 프로필별 독립 편집은 지원하지 않는다.
- `reportSections`, `checklistItems`, `briefingText`를 단일 편집 소스로 유지한다.
- 화면 토글은 `미리보기 + 내보내기 결과`에만 영향 준다.
- 단계 전이와 저장 상태 규칙은 기존 `AssessmentContext` 정의를 유지한다.

## 법령 기반 개선조치 출력 규칙
- `law-remedial-actions`는 법령 원문형 문장(`다음 각 호`, `사업주는`, 조문 번호 직접 인용)을 그대로 노출하지 않고 사용자 실행문으로 정제한다.
- 문장이 절단되었거나 조문체로 길어질 경우, 단계(`즉시/당일/재개 전/개선`)에 맞는 안전조치 문장으로 대체한다.
- `해야 합니다.해야 합니다.` 같은 중복 종결 표현은 단일 문장으로 정규화한다.
- 조치 문장과 근거 조문은 항상 분리 표기한다: `- [단계] 조치 문장 (근거: 법령명 제NN조(조문명))`.
