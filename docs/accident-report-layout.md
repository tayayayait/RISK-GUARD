# 산업재해조사표 레이아웃 기준

## 기준 원본
- PDF 원본: `public/forms/산업재해조사표.pdf`
- 법정 양식 원본(HWP): `legal-markdown/산업재해조사표양식.hwp`
- 구현 대상 컴포넌트: `src/components/forms/AccidentReportForm.tsx`
- 좌표 스키마: `src/components/forms/accidentReportLayout.ts`

## 렌더링 방식
- 기존 배경 이미지 방식은 사용하지 않는다.
- `pdfjs-dist`의 연산자 목록(`OperatorList`)을 파싱해 `SVG path + text`로 서식 선/채움/문구를 벡터 구조로 직접 렌더링한다.
- 입력 필드는 절대좌표(`x,y,w,h`) 오버레이로 배치한다.
- 좌표 기준 단위는 원본 PDF 페이지 좌표(`594.88 x 841`)다.
- 사고내용 입력(`FormEditor`의 상황 입력 textarea)은 내용 길이에 따라 높이가 자동 확장된다.
- 오버레이 멀티라인 필드(`재해발생 당시 상황`, `재해발생원인`, `재발방지계획`)는 입력 줄 수에 따라 높이가 자동 확장된다.
- 멀티라인 필드가 확장되면 하단 항목은 누적 오프셋으로 함께 내려가 행 겹침을 방지한다.
- 사고내용 기반 자동작성 시 AI 분석 결과를 바탕으로 아래 항목을 문서 형식으로 정리해 채운다.
  - 상해종류(질병명), 상해부위(질병부위), 발생장소, 재해관련 작업유형
  - 재해발생 당시 상황, 재해발생 원인, 재발방지계획
- 사고서식 자동작성(사고보고 전용) 규칙
  - 입력: `작업/사고 제목(선택)`, `현재 작업 상황/사고 발생 내용(필수)`
  - 출력: `재해관련 작업유형(당시 상황)`, `재해발생원인`, `재발방지계획` 3개 항목을 산업재해조사표 서술문 형태로 생성
  - 품질 게이트: 항목 간 중복 문장 제거, 원인 항목은 기여요인별 분리, 재발방지계획은 실행형 3개 항목(번호형)으로 정리

## 좌표/스케일 규칙
- 기준 페이지 크기
  - 폭: `594.88`
  - 높이: `841`
- 렌더 캔버스는 `aspect-ratio: 594.88 / 841` 고정
- 필드 스타일 계산식
  - `left = x / 594.88 * 100%`
  - `top = y / 841 * 100%`
  - `width = w / 594.88 * 100%`
  - `height = h / 841 * 100%`
- 체크박스도 동일하게 절대좌표로 배치한다.
- `발생일시`처럼 구분 칸 폭이 서로 다른 구간은 균등 grid를 사용하지 않고 칸별 절대좌표를 개별 정의한다.

## 폰트 및 입력 규칙
- 오버레이 필드 폰트 우선순위: `Dotum, Batang, Malgun Gothic, Noto Sans KR`
- 단일행 필드
  - 줄바꿈 없음, overflow clip
- 다중행 필드
  - `textarea` 기반, 줄바꿈 유지
  - 기본 높이를 초과하는 경우 자동 확장되며 스크롤 대신 행 높이 증가를 사용
- 체크 항목
  - 원본 서식의 `[ ]`는 배경에 포함된 것으로 간주하고, UI에서 기본 체크박스 박스를 추가 렌더하지 않는다.
  - 선택된 항목에만 지정 좌표에 `✓`를 오버레이한다.
  - 체크마크 좌표는 원본 PDF의 `[`와 `]` 사이 중심점 기준으로 관리한다.
- 고정값(읽기 전용)
  - 사업자등록번호
  - 산재관리번호
  - 사업장명
  - 업종
  - 소재지(본사)

## PDF 다운로드 규칙
- 구현 파일: `src/lib/accidentReportPdfBuilder.ts`
- 캡처 루트 DOM id: `accident-report-export-root`
- `법정서식(PDF) 다운로드`는 캡처 결과를 A4 전체면으로 저장한다.
- 무여백 고정
  - `x=0`, `y=0`
- `width=pageWidth`, `height=pageHeight`
- 폼 외 요소가 포함되지 않도록 캡처 루트 내부는 SVG 서식 + 오버레이 필드만 유지한다.

## DOCX 다운로드 규칙
- 구현 파일: `src/lib/accidentReportDocxBuilder.ts`
- `법정서식(DOCX) 다운로드`는 새 문서를 그리지 않고, `legal-markdown/산업재해조사표양식.hwp` 기준 템플릿(`public/forms/산업재해조사표양식.docx`)을 로드한 뒤 `word/document.xml`의 해당 셀만 치환한다.
- 템플릿 패키지의 스타일/표 경계/단락 구조는 유지하고, 사용자 입력값은 빈칸 셀에만 매핑한다.
- `재해발생 당시 상황`, `재해발생원인`, `재발방지계획`은 줄바꿈/문장 경계 기준으로 `w:p` 문단 단위로 채워 넣어, Word 열람 시 항목별 문단이 분리된 상태로 출력한다.
- 체크/일시 항목은 템플릿의 기존 슬롯(`[ ]`, `년/월/일/시/분` 고정 문구)을 유지한 채 슬롯 값만 치환한다.
- 템플릿 변환본에 표 경계선 메타가 누락된 경우를 대비해, DOCX 생성 시 `w:tblBorders`를 보강해 칸 경계가 손실되지 않도록 한다.
- DOM 캡처 이미지 삽입(`word/media/image1.png`)은 사용하지 않는다.
- 필드 매핑 대상은 `AccidentReportData` 전 항목(파견사업주 정보 포함)이다.
- enum/체크 항목(발주자, 국적, 고용형태, 근무형태, 사망 여부, 기술지원 요청, 개인정보 동의)은 `[✓]/[ ]` 표기로 채운다.

## 유지보수 원칙
- 서식 경계선/배경은 원본 PDF 기준을 우선한다.
- 새 필드 추가/조정 시 좌표 스키마만 수정하고, 임의 테이블 구조 변경은 금지한다.
- 주민등록번호는 `앞 7칸 + 뒷자리 1칸`을 개별 입력칸으로 분리해 각 칸 중앙 정렬을 유지한다.
- 발생일시는 `년/월/일/요일/시/분` 칸별 절대좌표를 유지하며, 자릿수 제한을 함께 관리한다.
- 하단 안내문 구역은 각 안내 문단과 선택 체크박스를 한 행(`flex`)으로 묶어 좌우 정렬한다.
- 하단 체크박스는 문단 우측 고정 박스(`border + bg`) 안에 배치해 줄바꿈 시에도 위치가 흔들리지 않게 유지한다.
- 작성자 정보(`성명/전화번호/작성일`)는 동일 행 그룹으로 정렬하고, 서명 영역(`사업장 대표/근로자 대표`)은 2열 블록으로 정렬한다.
- 레이아웃 변경 후에는 화면 비교와 PDF/DOCX 다운로드 결과를 같이 검증한다.
